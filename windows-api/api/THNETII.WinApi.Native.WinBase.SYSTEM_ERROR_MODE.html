Fields

Name Description
SEM_FAILCRITICALERRORS

The system does not display the critical-error-handler message box. Instead, the system sends the error to the calling thread.

Best practice is that all applications call the process-wide function with a parameter of SEM_FAILCRITICALERRORS at startup. This is to prevent error mode dialogs from hanging the application.
SEM_NOALIGNMENTFAULTEXCEPT

The system automatically fixes memory alignment faults and makes them invisible to the application. It does this for the calling process and any descendant processes. This feature is only supported by certain processor architectures. For more information, see .
SEM_NOGPFAULTERRORBOX

The system does not display the Windows Error Reporting dialog.
SEM_NOOPENFILEERRORBOX

The THNETII.WinApiNative.FileManagement.FileManagementFunctions.OpenFile function does not display a message box when it fails to find a file. Instead, the error is returned to the caller. This error mode overrides the flag.
