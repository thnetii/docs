ThrowIfNull<T>(T, String)

Checks whether the passed argument is null, throwing an appropiate ArgumentNullException if it is.

Declaration
public static T ThrowIfNull<T>(this T instance, string name)
    where T : class
Parameters
Type Name Description
T instance

The instance to check for null.
String name

The argument name as it appears in the calling method. Use the builtin nameof keyword.
Returns
Type Description
T

The instance argument, to allow for chained method calls.
Type Parameters
Name Description
T

The type of the argument. Must be a reference type.
Remarks

This is a simple convenience extension method for a common argument null check for reference types.

As this method is implemented as an extension method, it can be called with the dot operator on instance instance even if instance is null without triggering a NullReferenceException.

ThrowIfNullOrEmpty(String, String)

Checks whether a passed string argument is null or the empty string, and throws an appropiate ArgumentException if that is the case.

Declaration
public static string ThrowIfNullOrEmpty(this string value, string name)
Parameters
Type Name Description
String value

String name

The argument name as it appears in the calling method. Use the builtin nameof keyword.
Returns
Type Description
String

The value of value, to allow for chained method calls.
Remarks

In addition to calling IsNullOrEmpty(String), this method performs an additional null-reference check to throw a ArgumentNullException instead of the more general ArgumentException if value is null.

As this method is implemented as an extension method, it can be called with the dot operator on value instance even if value is null without triggering a NullReferenceException.

Returns
Type Description
IEnumerable<T>

A wrapper around enumerable enumerating the same elements.
Type Parameters
Name Description
T

The type of the elements enumerated by enumerable.
Remarks

In order to ensure that enumerable is not empty, the enumerable has to be expanded by checking if the first element is availble. To prevent double evaluation of the enumerable, the created enumerator is preserved and wrapped in the new enumerable that is returned.

Type Condition
ArgumentException

enumerable does not contain any elements.
